Zombie Congo Line

PROJECT DESCRIPTION

This project involves the creation of a Doubly-Linked List class (Sequence) and a Node class (SequenceNode), implemented in C++. Additionally, a Zombie class is created to represent the elements in the list. The project simulates a Zombie Conga Line, where each node in the linked list represents a dancing zombie. The colors of zombies (Red, Yellow, Green, Blue, Magenta, and Cyan) determine their type.

Key Features:

1. Dynamic Doubly-Linked List Implementation (Sequence Class): The backbone of the project, this class demonstrates a comprehensive application of a doubly-linked list, allowing for efficient insertion and deletion from both ends and at any index. The templated design ensures versatility for various data types.

2. Zombie Manipulation and Interaction: The core of the simulation, involving creating, adding, and removing Zombie instances from the list. This includes actions like adding zombies at the front (Engine), at the end (Caboose), or at a specific index (Jump in the Line).

3. Node Class Design (SequenceNode Class): A fundamental component that forms the nodes of the doubly-linked list. Each node's ability to link to its previous and next node allows for bidirectional traversal, crucial for the list's dynamic operations.

4. User-Driven Simulation and Output: The user can control the number of simulation rounds and choose to continue or end the party, making the program interactive. The list's state is outputted after each action, providing real-time feedback on the conga line's status.

Sun Devil Bank Account Management System

PROJECT DESCRIPTION

This project involves the development of a dynamic hash table-based system for managing bank account records at Sun Devil Bank. Each bank account record contains an account number, owner's name, and current balance. Unique six-digit account numbers ranging from 100001 to 999999 are assigned to each account. The system is designed to handle a maximum of 800 accounts at any given time. The implementation of the hash table is done in two distinct methods: Hashing with chaining for collision handling and Hashing with open addressing using linear probing.

Key Features:

1. Efficient Collision Handling: Implements two distinct methods for managing collisions in a hash table - hashing with chaining and hashing with open addressing using linear probing. This feature ensures that the system can handle a high number of accounts efficiently, even when multiple accounts hash to the same index.

2. Dynamic Account Record Management: Provides capabilities for inserting, deleting, and searching bank account records dynamically. This allows for real-time management of bank accounts, ensuring that the bank's database is always up to date.

3. Scalable Design for Account Handling: Despite the high range of possible account numbers (100001 to 999999), the system is designed to efficiently handle up to 800 accounts at any given time, demonstrating scalability and adaptability to the bank's needs.

4. User-Friendly Interface for Account Operations: The system includes a simple, interactive console interface that allows users to easily perform operations like inserting a new account, deleting an existing account, or searching for a specific account, making it accessible for bank staff with various levels of technical expertise.

AirTrafficBST: Efficient Plane Landing Scheduler

PROJECT DESCRIPTION

AirTrafficBST is a software solution designed to optimize the scheduling of airplane landings at airports with a single runway. Utilizing the principles of Binary Search Trees (BST), this system effectively manages landing requests while ensuring a predefined time gap (K) between each landing. It addresses the challenge of allocating runway time slots to incoming flights, considering existing schedules and the required time gap for safe operations. The application is particularly useful for controlling traffic in busy airports, ensuring both efficiency and safety in managing landing requests.

Key Features:

1. Gap Enforcement in Landing Requests: The system ensures that each new landing request maintains a minimum time gap (K) from other scheduled landings, which is crucial for maintaining safety standards on a single runway airport. This feature is vital for preventing runway overcrowding and ensuring safe intervals between landings.

2. Dynamic Landing Management: This feature allows for real-time management of landing schedules. It includes functionalities for inserting new landing requests and withdrawing existing ones, which is essential in accommodating the dynamic and often unpredictable nature of air traffic, such as delays or cancellations.

3. Efficient Scheduling with BST: The application leverages the efficiency of Binary Search Trees (BST) for the storage and retrieval of landing times and flight information. This enables fast and efficient scheduling operations, including insertion, removal, and search of landing times, which is key to managing a complex and frequently changing schedule.
